Integration with Editors and IDEs

  • Configure an Editor
  • Using an Editor
  • Installation Path Examples

Closure FX Builder is integrated with the most common editors and IDEs for JavaScript development. If the editor which you uses is configured, you can immediately open the issued Warnings and Errors in the current editor by one action.

  • Currently the tool is integrated with:
    • WebStorm (Windows, Unix, Mac OS)
    • Notepad++ (Windows)
    • Komodo Edit (Windows, Unix, Mac OS)
    • Sublime Text (Windows, Unix, Mac OS)
    • Emacs (Windows, Unix, Mac OS)
    • jEdit (Windows, Unix, Mac OS)
    • TextMate (Mac OS)

If your favorite editor is not in the list above, please submit Enhancement request.

Configure an Editor

  1. Click Settings button to open Closure FX Preferences dialog.
  2. Select Editors tab.

  3. Click Browse... button for one of the available editors.
  4. Choose the installation directory (Windows, Unix) or application file (Mac OS) of the editor.

    For example:
    on Windows: /Applications/Sublime Text 2.app
    on Unix: /home/john/Sublime Text 2
    on Windows: C:/Program Files/Sublime Text 2

  5. Click Mark as Default radiobox to use the chosen editor by default.

  6. Click Apply button to save.

Using an Editor

  1. Run your Build Configuration and go to the Console tab.
  2. If the compiler finds problems, just double-click a warning or an error to inspect the code in your favorite editor.

  3. Right-click a warning or an error to choose other editors from the context menu.
